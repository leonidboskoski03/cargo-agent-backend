# 40 - Doc Update Obligations

Keep docs and code synchronized in the same PR/change.

- API behavior change -> update `docs/contracts/api/*.md`.
- Capability/state change -> update `docs/context/implementation-status.md`.
- Product/domain boundary change -> update `docs/context/product-context.md` (and architecture if needed).
- Release risk/gates change -> update `docs/release/*.md`.
- Execution status change -> update `docs/delivery/roadmap.md`, active sprint file, and append to `docs/delivery/task-log.md`.
