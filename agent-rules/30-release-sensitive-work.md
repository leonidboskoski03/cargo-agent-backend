# 30 - Release-Sensitive Work

Release-sensitive areas include auth, OTP, sessions, invites, billing, webhooks, CI gates, and UAT.

- Load `docs/release/mvp-readiness.md` before changes.
- Use `docs/release/evidence-map.md` for gate/evidence requirements.
- Use `docs/release/runbook.md` for operational stop/fail rules.
- Use `docs/release/uat-smoke-checklist.md` and `docs/release/go-no-go.md` for validation and decision basis.
- If risk posture changes, update release docs in the same change.
